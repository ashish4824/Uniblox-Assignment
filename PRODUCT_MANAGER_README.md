# Product Manager Assignment

##  Assignment Overview

This is a Product Management take-home assignment focused on defining and prioritizing AI-powered features for a document intelligence platform. The assignment tests product thinking, prioritization skills, technical feasibility assessment, and communication abilities.

##  Scenario

You are a Product Manager at a fast-growing AI company building next-generation productivity tools. Your team has developed an AI-powered document intelligence platform with a new LLM and reasoning engine capable of:

1. **Extracting structured data** from unstructured documents (emails, PDFs, contracts, reports)
2. **AI-driven summarization** and recommendations based on extracted insights
3. **Automated decision-making** (risk assessment, approvals, prioritization)

Your task is to transform this AI innovation into a real product feature that delivers user value.

---

##  Assignment Structure

### Part 1: Feature PRD/User Story

Write a Product Requirements Document (PRD) for a new AI-powered feature including:

#### 1. Problem Statement
- What user pain point does this feature solve?
- Why is this problem worth solving?
- What is the impact of not solving it?

#### 2. User Persona
- Who are the primary users?
- What are their goals and challenges?
- How do they currently handle this problem?
- What are their pain points with current solutions?

#### 3. Proposed Solution
- How does the AI-powered feature work?
- How does it integrate into the platform?
- What is the user experience flow?
- What makes this solution better than alternatives?

#### 4. Key Functional Requirements
Must-have functionalities (use bullet points or user stories):
- As a [user type], I want to [action] so that [benefit]
- Core features and capabilities
- Integration requirements
- Performance requirements

#### 5. Success Metrics
How will we measure impact?
- Primary metrics (North Star metric)
- Secondary metrics
- Counter metrics (what could go wrong)
- Leading indicators vs lagging indicators

#### 6. Potential Risks & Mitigations
- Technical risks and solutions
- Business risks and mitigations
- User adoption risks
- Competitive risks

---

### Part 2: Prioritization & Scope Definition

**Three Features to Consider:**

1. **AI-Powered Document Summarization**
   - Automatically generates concise summaries
   - Target: Long reports, contracts, meeting notes

2. **Automated Data Extraction & Structuring**
   - Converts unstructured to structured data
   - Target: Invoices, resumes, customer feedback

3. **AI-Driven Workflow Automation**
   - Uses reasoning to automate decisions
   - Target: Approval/rejection workflows

**Answer the Following:**

#### Which Feature to Prioritize First?
- Use a prioritization framework (RICE, Value vs Effort, etc.)
- Consider: User value, business impact, technical feasibility, strategic alignment
- Provide clear rationale for your choice

#### MVP Scope Definition
- What is the Minimum Viable Product?
- What features are in scope vs out of scope?
- What are the acceptance criteria?
- What does "done" look like?

#### Iterative Next Versions
- Version 1 (MVP): Core functionality
- Version 2: Enhanced capabilities
- Version 3: Advanced features
- Long-term vision

#### Data-Driven Validation
- What data would you collect to validate prioritization?
- What experiments or user research would you conduct?
- What would change your decision?
- How would you measure success?

### Part 3: AI Feasibility & Go-To-Market Plan

#### Technical & Business Considerations

**Technical:**
- Model accuracy and reliability requirements
- Latency and performance expectations
- Data privacy and security requirements
- Scalability considerations
- Integration complexity

**Business:**
- Market readiness and timing
- Competitive landscape
- Pricing and monetization strategy
- Resource requirements
- ROI expectations

#### AI Model Validation & Refinement

**Pre-launch Validation:**
- Beta testing strategy
- A/B testing approach
- User feedback loops
- Quality assurance process
- Edge case handling

**Refinement Process:**
- How to collect user feedback
- Metrics to monitor
- Iteration cycles
- Human-in-the-loop considerations
- Continuous improvement plan

#### User Adoption & KPIs

**Adoption Strategy:**
- User onboarding plan
- Change management
- Training and documentation
- Communication plan
- Incentives for early adopters

**Post-Launch KPIs:**
- Adoption metrics (DAU, MAU, activation rate)
- Engagement metrics (frequency, depth)
- Satisfaction metrics (NPS, CSAT)
- Business metrics (revenue, retention)
- Product-specific metrics (accuracy, task completion)
